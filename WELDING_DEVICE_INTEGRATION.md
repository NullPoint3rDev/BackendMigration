# 🔧 Интеграция со сварочными аппаратами

## Обзор

Система WeldTelecom теперь поддерживает подключение к сварочным аппаратам через TCP/IP соединения. Реализована полная архитектура для мониторинга и управления аппаратами в реальном времени.

## 🏗️ Архитектура

### Компоненты системы:

1. **TcpDeviceClient** - TCP-клиент для подключения к сварочным аппаратам
2. **WeldingDataParserService** - парсер данных от аппаратов
3. **WeldingMachineStateService** - сервис для сохранения состояний в БД
4. **WeldingDeviceManagerService** - менеджер для управления множественными аппаратами
5. **DeviceController** - WebSocket контроллер для real-time обновлений
6. **WeldingDeviceController** - REST API для управления аппаратами

## 📡 Протокол обмена данными

### Формат входящих сообщений:
```
:MAC_ADDRESS;DATA_PAYLOAD
```

**Пример:**
```
:8CAAB579425A;0102030405060708090A0B0C0D0E0F
```

**Структура:**
- `:` - начало сообщения
- `MAC_ADDRESS` - 12-символьный MAC-адрес аппарата
- `;` - разделитель
- `DATA_PAYLOAD` - данные состояния аппарата

### Парсинг данных:
- **Позиции 0-1**: Модель аппарата
- **Позиции 2-3**: Версия прошивки
- **Позиции 4-5**: Материал сварочной проволоки
- **Позиции 6-7**: Ток (А)
- **Позиции 8-9**: Напряжение (В)
- **Позиции 10-11**: Расход газа (л/мин)
- **Позиции 12-13**: Температура (°C)
- **Позиции 14-15**: Статус управления

## 🌐 API Endpoints

### REST API для управления аппаратами:

#### Получить состояние аппарата
```
GET /api/welding-devices/{mac}/state
```

#### Получить статус подключения
```
GET /api/welding-devices/{mac}/connection
```

#### Получить состояния всех аппаратов
```
GET /api/welding-devices/states
```

#### Получить статистику
```
GET /api/welding-devices/statistics
```

#### Отметить аппарат как отключенный
```
POST /api/welding-devices/{mac}/disconnect
```

### WebSocket для real-time обновлений:

#### Подключение
```
WebSocket: /api/ws
```

#### Топики:
- `/topic/device-state` - обновления состояния аппаратов
- `/topic/device` - сырые данные от аппаратов

## 📊 Статусы аппаратов

### WeldingMachineStatus:
- **Offline** - аппарат отключен
- **Online** - аппарат подключен
- **Idle** - аппарат в режиме ожидания
- **Welding** - аппарат работает (сварка)
- **Error** - ошибка на аппарате
- **Maintenance** - техническое обслуживание
- **Calibration** - калибровка
- **Testing** - тестирование

## 🔧 Настройка для новых аппаратов

### 1. Добавить аппарат в конфигурацию:
```yaml
welding:
  devices:
    - mac: NEW_MAC_ADDRESS
      type: NEW_TYPE
      ip: NEW_IP_ADDRESS
      port: 3000
      description: Описание аппарата
```

### 2. Создать запись в базе данных:
```sql
INSERT INTO "WeldingMachine" (
    "OrganizationUnitID", 
    "WeldingMachineTypeID", 
    "Status", 
    "Name", 
    "MAC"

    1,  -- ID подразделения
    1,  -- ID типа аппарата

    'NEW_MAC_ADDRESS'
);
```

## 🛠️ Устранение неполадок

### Проблема: TCP-клиент не подключается
**Решение:**
1. Проверить IP и порт в конфигурации
2. Убедиться, что сварочный аппарат доступен по сети
3. Проверить файрвол

### Проблема: Данные не парсятся
**Решение:**
1. Проверить формат данных от аппарата
2. Настроить парсер в `WeldingDataParserService`
3. Проверить логи парсинга

### Проблема: WebSocket не работает
**Решение:**
1. Проверить конфигурацию WebSocket в `WebSocketConfig`
2. Убедиться, что фронтенд подключается к правильному URL
3. Проверить CORS настройки
