## Шаги для настройки внешнего доступа

### 1. Получение внешнего IP адреса

```bash
# На macOS/Linux
curl ifconfig.me
# или
curl ipinfo.io/ip
```

### 2. Настройка роутера (если приложение на домашнем компьютере)

1. Войдите в настройки роутера (обычно 192.168.1.1 или 192.168.0.1)
2. Найдите раздел "Port Forwarding" или "Проброс портов"
3. Добавьте правила для портов:
   - **Порт 8083** → ваш локальный IP (для backend)
   - **Порт 3001** → ваш локальный IP (для frontend)

### 3. Настройка файрвола

#### macOS:
```bash
# Разрешить входящие подключения на порты
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /usr/bin/java
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /usr/local/bin/node
```

#### Linux (Ubuntu/Debian):
```bash
sudo ufw allow 8083
sudo ufw allow 3001
```

### 4. Запуск приложения

#### Вариант 1: Запуск всего приложения одной командой
```bash
./start-all.sh
```

#### Вариант 2: Запуск по отдельности
```bash
# Терминал 1 - Backend
./start-backend.sh

# Терминал 2 - Frontend
./start-frontend.sh
```

#### Вариант 3: Ручной запуск
```bash
# Backend
cd WT2
./mvnw spring-boot:run

# Frontend (в другом терминале)
cd BetaFrontWT
npm run start-external
```

## Доступ к приложению

После настройки приложение будет доступно по следующим адресам:

- **Frontend**: `http://YOUR_EXTERNAL_IP:3001`
- **Backend API**: `http://YOUR_EXTERNAL_IP:8083/api`

Замените `YOUR_EXTERNAL_IP` на ваш внешний IP адрес.

## Проверка доступности

### Проверка backend:
```bash
curl http://YOUR_EXTERNAL_IP:8083/api/health
```

### Проверка frontend:
```bash
curl http://YOUR_EXTERNAL_IP:3001
```

## Безопасность

⚠️ **Важно**: Открытие портов для внешнего доступа создает потенциальные риски безопасности:

1. **Нужно использовать HTTPS в продакшене**
2. **Нужно настроить аутентификацию для доступа к API :: Сделано**
3. **Нужно ограничить доступ только необходимыми IP адресами**
4. **Нужно регулярно обновлять зависимости**
5. **Нужно мониторить логи на подозрительную активность**

## Устранение проблем

### Проблема: "Connection refused"
- Проверьте, что приложение запущено
- Убедитесь, что порты открыты в файрволе
- Проверьте настройки роутера

### Проблема: CORS ошибки
- Убедитесь, что backend запущен на правильном порту
- Проверьте настройки CORS в WebConfig.java

### Проблема: "Cannot connect to database"
- Убедитесь, что PostgreSQL доступен
- Проверьте настройки подключения в application.yml
